## Implementation Decisions

### Структура данных порогов
- **Массив пар:** `[{from: 0, to: 25}, {from: 0, to: 60}]`
- **Тип:** `ThresholdPair = { from: number; to: number }`
- **Причина:** Проще и понятнее, легко добавлять/удалять через array methods. ID не нужны пока нет необходимости ссылаться на конкретные пороги из других мест.

### UI для управления порогами
- **Расширяемый список с кнопками добавить/удалить**
- **Каждая строка:** поля "от" и "до" + кнопка удалить
- **Кнопка добавить:** добавляет новую пустую пару в конец списка
- **Причина:** Интуитивно, соответствует паттерну управления списками в UI

### Детекция с несколькими порогами
- **Один проход по данным:** проверяем каждую точку скорости на все пороги одновременно
- **Алгоритм:** Для каждой точки данных проверяем пересечение с каждым порогом в массиве
- **Производительность:** O(n) вместо O(n*m) где m — количество порогов
- **Причина:** Более эффективно для больших CSV файлов

### Отображение в таблице
- **Да, показывать какой порог использовался для каждой попытки**
- **Формат:** Шаблоны переключения между скоростями (0-25, 0-60, 0-90, 0-100) и кастомные
- **Колонка:** "Порог" или "Threshold" в таблице AccelerationTable
- **Причина:** Пользователь должен знать для какого порога найдена каждая попытка

### Выбор порога в визуализации
- **Пресеты + список:**
  - Пресет кнопки для стандартных наборов (как сейчас: 0-25, 0-60, 0-90, 0-100, custom)
  - Для custom — выпадающий список с названиями порогов ("Порог 1: 0-25", "Порог 2: 30-100")
  - Или кнопки рядом с графиком для каждого кастомного порога
- **Причина:** Сохраняет существующий паттерн пресетов + добавляет гибкость для кастомных порогов

### Пресеты для наборов порогов
- **Отложены** (план 5.8 optional)
- **Причина:** Можно добавить в будущей фазе. Сначала реализуем базовую функциональность.

### Claude's Discretion
- **Детали реализации UI:** Точный стиль списка порогов (как TripOverview settings panel или другой паттерн)
- **Валидация пар:** Как обрабатывать перекрывающиеся или идентичные пары
- **Именование кастомных порогов:** Автоматическая генерация ("Порог 1", "Порог 2") или пользовательские имена
- **Порядок сортировки:** Сохранять порядок добавления или сортировать по значению

## Specific Ideas

- Структура: `const thresholdPairs: ThresholdPair[] = [{ from: 0, to: 25 }, { from: 0, to: 60 }]`
- detectAccelerations: Принимает `thresholdPairs: ThresholdPair[]` вместо `targetSpeed: number`
- Возвращает: `Map<ThresholdPair, AccelerationAttempt[]>` или плоский массив с полем thresholdPair
- AccelerationAttempt добавить поле: `thresholdPair?: { from: number; to: number }`
- UI списка: похож на настройки TripOverview (glassmorphism, кнопки добавить/удалить)
- Пресеты в визуализации: сохраняем существующие кнопки (0-25, 0-60, 0-90, 0-100, custom)
